## Architecture

This portfolio is a **client-rendered React SPA** built on top of Vite. There is no backend in this repository; all server‑side concerns (hosting, SSL, CDN, etc.) are delegated to the deployment platform (e.g., Vercel).

### Top-Level Overview

- **Entry point**: `src/main.jsx`
  - Mounts React into `#root`
  - Imports base styles from `src/styles/base.css` and the main theme from `src/styles.css`
  - Wraps the app in `HashRouter` (or `BrowserRouter` if configured)
- **Application shell**: `src/App.jsx`
  - Declares all routes via `<Routes>` / `<Route>`
  - Delegates layout (navbar, background, footer) to `Layout`
- **Layout**: `src/components/Layout.jsx`
  - Uses `useLocation` to scroll to top on route change
  - Applies global effects (`useCursorEffect`, `useScrollAnimations`)
  - Renders `Background`, `Navbar`, route content, and `Footer`

```text
index.html
  └─ src/main.jsx
       └─ <Router>
            └─ <Layout>
                 ├─ <Background />
                 ├─ <Navbar />
                 ├─ <Routes> (pages)
                 └─ <Footer />
```

### Routing

Defined in `src/App.jsx` using React Router DOM:

- `/` → `Home`
- `/about` → `About`
- `/projects` → `Projects`
- `/projects/*` → Project detail pages (ChessLytics, SpotiFriend, NLP, Movie, ETL tools, EEG research)
- `/hobbies/*` → Hobby pages (Piano, Chess, Volleyball, Skateboarding)
- `/resume` → `Resume`
- `/contact` → `Contact`
- `/contact-success` → `ContactSuccess`

Router type:
- **HashRouter** in `main.jsx` for GitHub Pages compatibility
- Can be switched to **BrowserRouter** when deploying to Vercel/Netlify (with proper rewrite rules)

### Components and Pages

- `src/components/`
  - `Navbar` – top navigation with active state and mobile menu
  - `Footer` – simple site footer
  - `Layout` – shared shell around all routes
  - `Background` – layered animated background / particles
- `src/pages/`
  - `Home`, `About`, `Projects`, `Resume`, `Contact`, `ContactSuccess`
  - `pages/hobbies/*` – hobby pages with copy + media
  - `pages/projects/*` – project detail pages; some are structured placeholders for future content migration

### Hooks / Effects

- `useCursorEffect`
  - Adds a custom cursor + trailing elements
  - Handles hiding the custom cursor over iframes / PDFs
  - Scales cursor slightly on interactive elements

- `useScrollAnimations`
  - Uses IntersectionObserver on `section` elements
  - Adds utility classes (e.g., `fade-in`, `visible`) to trigger CSS animations as sections enter the viewport

### Assets and Build Output

- **Source images**: `public/static/images/`
  - Organized by feature: `home/`, `about/`, `projects/*`, `hobbies/*`, `resume/`, `shared/`
- **Build output**: `dist/`
  - Generated by `vite build`
  - Contains `index.html`, hashed JS/CSS bundles, and copied assets (`dist/static/images/...`)

### Third-Party Integrations

- **Formspree**: Contact form submission (`src/pages/Contact.jsx`)
  - `fetch` POST to Formspree endpoint
  - On success → navigate to `/contact-success`
- **YouTube embeds**: Various hobby pages (especially Piano)
  - `<iframe>` embeds for performances

### Environment and Config

- `vite.config.js`
  - `plugins: [react()]`
  - `base: '/'` (root deployment; can be changed for sub-path hosting)
  - `build.outDir = 'dist'`, `build.assetsDir = 'assets'`

There is intentionally **no backend** or server-side rendering. All behavior is client-side and compatible with static hosting/CDN deployment.
